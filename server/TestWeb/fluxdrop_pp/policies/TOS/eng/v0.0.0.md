## 3. Storage and Quotas

### 3.1 Storage Allocation

Each user account is allocated a storage quota. The default quota is **50 GB** and adjusts dynamically based on server capacity:

| Server free space | Effective quota |
|---|---|
| ≥ 500 GB | 100 GB |
| 200 – 500 GB | 50 GB (default) |
| 100 – 200 GB | 35 GB |
| < 100 GB | 10 GB |

Your current quota and usage are always visible in your profile. Trash bin contents do **not** count against your quota.

### 3.2 Trash Bin and File Retention

When you delete a file it is moved to a trash bin rather than being removed immediately.

- **Standard retention**: 30 days. After this period the file is permanently deleted during routine maintenance.
- **Reduced retention**: 7 days. Applied automatically when server disk space is critically low. You will see a notice in the trash bin view when this is in effect. Retention returns to 30 days once space is freed.

Files permanently deleted from the trash bin are removed from user access immediately. Physical overwriting of disk blocks occurs during routine server maintenance cycles.

### 3.3 Shared Links and Expiry

You may generate public share links for files and folders. Links may be configured with:

- An optional expiry date after which the link stops working
- A restriction requiring the visitor to hold a FluxDrop account
- Optional anonymous or authenticated uploads into the shared folder
- An optional inline preview mode

Share links that have expired are no longer accessible but the underlying files are not deleted unless you delete them manually.
