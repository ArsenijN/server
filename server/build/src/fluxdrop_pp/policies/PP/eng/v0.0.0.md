# FluxDrop Privacy Policy

**Version 0.0 — Last Updated: April 22, 2026**

---

## 1. Introduction

This Privacy Policy describes how FluxDrop, operated by ArsenijN, collects, stores, and uses information about you when you use the Service. By using FluxDrop you agree to the practices described here.

---

## 2. What We Collect

### 2.1 Account Information

When you register, we collect and store:

- **Username** — your login name
- **Nickname** — your publicly visible display name
- **Email address** — used for account verification
- **Password hash** — your password is processed using bcrypt, a one-way hashing algorithm. We store only the resulting hash and never the original password. It is not possible to recover your password from what we store.

### 2.2 Session Data

When you log in, we create a session. We store a **hashed** version of your session token in the database (the raw token is sent only to you). Sessions expire after 7 days.

### 2.3 Server Logs

FluxDrop maintains server logs that record, for each request:

- Timestamp of the operation
- IP address of the requester
- The full URL path accessed (e.g. `/api/v1/list/some-folder`)

URL paths are logged in full because this is necessary to diagnose errors and understand why issues occur. These logs are **not accessible from the internet** and are only readable by the server administrator.

### 2.4 File Metadata

We store metadata about your uploaded files, including file names, sizes, modification times, and share link settings you configure.

### 2.5 Upload and Download Activity

We log upload sessions and, for share links with statistics enabled, access counts and timestamps. We also store short-lived download tokens (hashed) to support resumable downloads.

## 3. How We Use Your Information

- **Authentication and security**: verifying your identity, rate-limiting requests, detecting abuse
- **Service delivery**: storing and serving your files, processing uploads, managing your trash bin and share links
- **Diagnostics**: investigating errors and service issues using server logs
- **Legal compliance**: responding to lawful requests from authorities (see Section 5)

---

## 4. Data We Do Not Collect

- We do not use third-party analytics or tracking services
- We do not serve advertisements and do not share your data with advertising networks
- We do not use cookies beyond what is necessary for session management
- We do not collect device fingerprints, browser histories, or any information beyond what is described in Section 2

Your data is not shared with any other party under any other circumstance.

---

## 6. Data Storage and Security

### 6.1 Passwords

Passwords are hashed using **bcrypt** with a per-password salt. We cannot recover your original password. If you forget it, it cannot be retrieved — only reset.

### 6.2 Session Tokens

Session tokens are stored in the database as SHA-256 hashes. The raw token exists only in your browser. A compromised database does not expose raw session tokens.

### 6.3 Server Infrastructure

The server and database are hosted on infrastructure accessible only to the administrator. Database files are not exposed to the internet. Log files are stored locally on the server.

---

## 7. Data Retention

### 7.1 Active Files

Your files are retained for as long as your account exists and the files have not been deleted.

### 7.2 Trash Bin

Deleted files are moved to a trash bin:

- **Standard retention**: 30 days, after which the file is permanently deleted during routine maintenance
- **Reduced retention**: 7 days, applied automatically when server disk space falls below a critical threshold. A notice is displayed in the trash bin view when this is in effect.

### 7.3 After Permanent Deletion

Once a file is permanently deleted it is removed from all user-facing access immediately. The physical disk blocks are overwritten during routine server maintenance cycles. Permanently deleted files may remain on the physical disk until that point.

### 7.4 Server Logs

Server logs are maintained for operational and security purposes. They are not automatically purged on a fixed schedule; retention is at the discretion of the administrator.

### 7.5 Account Deletion

If your account is deleted, your files and associated data are marked for deletion. Complete removal, including physical overwriting, follows the same process as standard file deletion.
